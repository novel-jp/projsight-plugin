# /learn-pj-execute — 実装フェーズ（07-02、必修）

07-01 で立ち上げたミニプロジェクトのタスクを自律的に遂行する演習です。

**所要時間**: 約 180 分
**前提**: `/learn-pj-kickoff`（07-01）完了済み
**スキル対応**: 統合演習（全スキル横断）

---

## Step 1: 導入

```
「実装フェーズです。07-01 で分解したタスクを順番に遂行してください。

ルール:
- 各タスクは start_work → 実装 → complete_work のフローで進める
- ブランチは task/<number>-<slug> の命名規則に従う（06-01）
- 中断する場合は upsert_task(notes) で状態を記録する（02-05）
- ブロッカーが発生したら upsert_issue で記録する

私はガイドしません。自律的に進めてください。
進捗確認や質問があればいつでも聞いてください。」
```

---

## Step 2: タスク一覧の確認

`list_tasks(projectId)` で 07-01 で作成したタスクの一覧を表示する。

```
「まず、どのタスクから着手しますか？
依存関係（blocks リンク）を考慮して順番を決めてください。」
```

---

## Step 3: 自律的な実装

受講者が各タスクを遂行する。スキルの役割は **最小限のガイド** のみ:

### 各タスクの遂行フロー

1. **開始**: `start_work(taskId)` で作業開始
2. **ブランチ作成**: `git checkout -b task/<number>-<slug>`
3. **実装**: Claude Code を活用してコードを書く
4. **中間チェックポイント**: 大きな区切りで `upsert_task(progressPct)` を更新
5. **コミット**: 適切なプレフィックスでコミット

### 受講者が詰まった場合のみ介入

- 「何から始めればいいか分からない」→ タスクの description を一緒に確認
- 「エラーが解決しない」→ 05-03（デバッグ）のスキルを想起させる
- 「設計を変更したい」→ DR を更新するか新規作成するか判断を促す

### 中断・再開の体験

最低 1 回は意図的に中断・再開を体験してもらう:

```
「ここで一度作業を中断しましょう。
02-05 で学んだように、upsert_task(notes) で次のセッションの自分に引き継ぎ情報を残してください。
notes には以下を含めましょう:
- 完了した作業の要約
- 次にやるべきこと
- 未解決の問題」
```

中断後、`start_work` で再開し、notes を参考に作業を続行する。

---

## Step 4: PR 作成

実装が完了したタスクについて PR を作成する:

```
「実装が完了しましたね。PR を作成しましょう。
06-02 で学んだフローを実践してください:
1. gh pr create で PR を作成
2. attach_pr でタスクに紐づけ
3. upsert_task(status: 'in_review') でレビュー待ちに」
```

受講者が学習用リポジトリを持っている場合は実際に PR を作成する。
持っていない場合は、ローカルコミットまでで完了とし、フローの説明で代替する。

---

## Step 5: タスク完了

各タスクを `complete_work(taskId)` で完了する。

```
「タスクを complete_work で完了しましょう。
成果物の completionPct が自動で更新されます。」
```

---

## Step 6: 実装フェーズの振り返り

全タスク（または時間内に完了できたタスク）が終わったら:

```
「実装フェーズお疲れさまでした！

振り返りポイント:
- start_work → complete_work のフローはスムーズでしたか？
- 中断・再開で notes は役に立ちましたか？
- ブロッカーや予想外の問題はありましたか？
- AI との協働で気づいたことはありますか？

全タスクを完了できなかった場合も心配いりません。
実際のプロジェクトでも、計画通りに進まないことは日常です。
大事なのは ProjSight に記録が残っていること — これが次に活きます。

次は:
- /learn-pj-incident（07-03）で障害対応シミュレーションに挑戦できます
- または /learn-pj-retrospect（07-04）で全体の振り返りに進めます」
```

受講者のタスクを `complete_work(taskId)` で完了にする。
