# /learn-pj-retrospect — 振り返り・評価（07-04、必修）

カリキュラム全体を振り返り、4+2 スキルの成長を自己評価する最終演習です。

**所要時間**: 約 60 分
**前提**: `/learn-pj-execute`（07-02）完了済み
**スキル対応**: 統合演習（メタ認知・自己評価）

---

## Step 1: 導入

```
「最終演習 — カリキュラム全体の振り返りです。

ProjSight に蓄積された記録を見返しながら、
あなたの成長を可視化し、今後の強化ポイントを明確にしましょう。

『どれだけ覚えたか』ではなく『どれだけ実践できるようになったか』が評価軸です。」
```

---

## Step 2: ProjSight の記録を振り返る

`list_tasks`, `list_drs`, `list_risks`, `list_issues` でプロジェクトの記録を一覧する。

```
「まず、あなたの学習記録を振り返りましょう。」
```

以下のサマリーを表示する:

- **タスク**: 完了数 / 全体数、completionPct
- **DR**: 作成した DR 一覧
- **リスク**: 登録したリスクとステータス
- **Issue**: 起票した Issue とステータス
- **成果物**: 進捗一覧

---

## Step 3: 4+2 スキルの自己評価

各スキルについて受講者に自己評価（1〜5）をしてもらう:

```
「各スキルについて、1〜5 で自己評価してください。
1 = まだ不安、3 = 基本はできる、5 = 自信がある

評価の参考にしてください:
```

| スキル           | 評価観点                                                   |
| ---------------- | ---------------------------------------------------------- |
| **Prompt Craft** | タスク記述の品質。01-01 の最初と比べて成長を感じるか       |
| **Context Eng.** | CLAUDE.md / DD の設計力。AI に適切な情報環境を提供できるか |
| **Intent Eng.**  | DR の品質。代替案・判断理由を自然に書けるか                |
| **Spec Eng.**    | タスク分解の適切さ。完了定義が検証可能か                   |
| **AI Collab**    | AI 出力の評価力。デバッグ時の情報構造化                    |
| **Git Workflow** | ブランチ・PR・CI のフローが自然に回せるか                  |

受講者が各スキルを評価したら、結果をテーブル形式で整理する。

---

## Step 4: タスク記述品質の成長比較

最初の演習（01-01 または 01-02）で書いたタスクと、07-01/07-02 で書いたタスクの品質を比較する。

```
「成長を可視化するために、最初に書いたタスクと最近書いたタスクを比較しましょう。」
```

1. 初期のタスクを `list_tasks` から特定
2. 07-01/07-02 のタスクを特定
3. 両方のタスクを `/review-task` スキルの観点（背景・対応内容・完了条件・制約）で比較
4. 具体的な改善点を指摘する

```
「初期のタスク記述:
{初期タスクの要約}

最近のタスク記述:
{最近タスクの要約}

改善されたポイント:
- {具体的な改善点}」
```

---

## Step 5: 改善計画

```
「自己評価を踏まえ、次に強化したいスキルはどれですか？

発展コース（各セクションの 02〜05）で重点的に練習できます:

| 強化したいスキル | 推奨演習                                          |
| ---------------- | ------------------------------------------------- |
| Prompt Craft     | /learn-pc-rewrite, /learn-pc-write                |
| Context Eng.     | /learn-ce-design, /learn-ce-dd, /learn-ce-maintain |
| Intent Eng.      | /learn-ie-write, /learn-ie-constraints            |
| Spec Eng.        | /learn-se-workflow, /learn-se-risk                |
| AI Collab        | /learn-ac-iterate, /learn-ac-debug                |
| Git Workflow     | /learn-gw-pr, /learn-gw-review                   |
```

受講者が選んだスキルに応じて、具体的な演習を推薦する。

---

## Step 6: 完了メッセージ

```
「カリキュラムを完了しました！おめでとうございます！

ProjSight の記録があなたの学習の軌跡です。
振り返りたいときはいつでも記録を見返せます。

学んだ 6 つのスキルは、AI 時代のエンジニアリングの基盤です:

  Prompt Craft — 明確な指示を書く
  Context Eng. — AI の情報環境を設計する
  Intent Eng. — 目的と判断基準をエンコードする
  Spec Eng. — 自律実行できる仕様を構築する
  AI Collab — AI の出力を評価・活用する
  Git Workflow — チーム開発のフローを回す

これからの業務でも、同じワークフローで
AI と協働しながらプロジェクトを推進してください。

/learn で全体の進捗をいつでも確認できます。」
```

受講者のタスクを `complete_work(taskId)` で完了にする。
