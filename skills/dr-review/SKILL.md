# /dr-review

プロジェクトの全 DR（Decision Record）を棚卸しし、ステータスの整理・クリーンアップを行う。

---

## 前提

- ProjSight MCP Server のツールを使用する
- ステータス変更はすべてユーザーに提案し、**承認を得てから** 実行する
- 一括変更ではなく、カテゴリごとに提案→承認→実行のサイクルで進める

---

## Step 1: DR 一覧の取得と分類

1. `list_drs(projectId)` で全 DR を取得
2. 以下のカテゴリに分類してテーブル表示する:

| カテゴリ             | 条件                       | アクション候補                                      |
| -------------------- | -------------------------- | --------------------------------------------------- |
| **要クリーンアップ** | `deprecated` ステータス    | close（完全に不要）or そのまま（参考として残す）    |
| **確認必要**         | `proposed` ステータス      | accepted（採用）/ deprecated（却下）/ close（不要） |
| **差し替え済み**     | `superseded` ステータス    | 後継 DR の確認、close 可否の判断                    |
| **現行有効**         | `accepted` / `implemented` | 現在も有効か確認（技術変更で陳腐化していないか）    |

---

## Step 2: カテゴリ別レビュー

各カテゴリについて以下を実行する:

### 2a. 要クリーンアップ（deprecated）

1. deprecated DR の一覧をテーブル表示（number, title, 理由）
2. 各 DR について以下を提案:
   - **close**: 参照価値がなく、完全に不要
   - **そのまま**: deprecated のまま参考情報として残す
3. ユーザーの承認を得てから `upsert_dr(drId, status)` を実行

### 2b. 確認必要（proposed）

1. proposed DR の一覧をテーブル表示（number, title, 作成日）
2. 作成から長期間経過（目安: 2フェーズ以上前）のものを特に注目
3. 各 DR について以下を提案:
   - **accepted**: 方針として採用されており、ステータスを正式化
   - **deprecated**: 方針が変わり、もはや不要
   - **そのまま**: 検討中のため保留
4. ユーザーの承認を得てから実行

### 2c. 差し替え済み（superseded）

1. superseded DR の一覧を表示
2. 後継 DR が存在し、内容が完全にカバーされているか確認
3. 参照価値がないものは close を提案

### 2d. 現行有効（accepted / implemented）

1. 全件レビューは不要。以下に該当するもののみ抽出:
   - 技術スタックの変更で陳腐化した可能性があるもの
   - 直近のフェーズで大きく方針が変わった領域のもの
2. 該当があれば deprecated or superseded への変更を提案

---

## Step 3: サマリ報告

レビュー完了後、以下のサマリをユーザーに報告:

```
## DR 棚卸し結果

- 総 DR 数: N 件
- 変更: close N件 / deprecated N件 / accepted N件 / その他 N件
- 変更なし: N件
- 現行有効: N件（accepted + implemented）

### 注意事項
[棚卸し中に発見した課題や、次回確認すべき事項があれば記載]
```

---

## 実行タイミング

| タイミング                 | 推奨度   |
| -------------------------- | -------- |
| フェーズ切替時（fork 後）  | **推奨** |
| DR が 10件以上増加した時   | 推奨     |
| 技術スタックの大きな変更後 | 推奨     |
| ユーザーから依頼された時   | 必須     |
