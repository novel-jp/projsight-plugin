# /learn-pc-reporting — ホウレンソウへの応用（01-05）

Prompt Craft を対人コミュニケーション（報告・連絡・相談）に応用する演習です。

**所要時間**: 約 30 分
**前提**: `/learn-pc-review` 完了推奨（必修ではない）
**スキル対応**: Prompt Craft（指示の構造化）

---

## Step 1: 導入

```
「Prompt Craft は AI への指示だけでなく、
人間へのコミュニケーションにも応用できます。

この演習では、報告・連絡・相談（ホウレンソウ）に

```

---

## Step 2: 悪い報告 vs 良い報告の比較

```
「まず、2 つの報告例を比較してみましょう。」
```

### 悪い報告

> すみません、例の件なんですが、うまくいかなくて...
> ちょっと遅れそうです。どうしましょう？

```
「この報告の問題点は何ですか？」
```

受講者の回答を待ってからフィードバック:

- 「例の件」→ 何の件か不明
- 「うまくいかない」→ 何がどう失敗しているか不明
- 「ちょっと遅れそう」→ どのくらい遅れるか不明
- 「どうしましょう」→ 対応案がなく、相手に丸投げ

### 良い報告

```markdown
## 状況

タスク #42 の API 実装で、外部サービス（Stripe API）のレート制限に
当たり、バッチ処理が想定の 3 倍の時間がかかっています。

## 影響

当初の完了予定（金曜日）に対して、1 営業日の遅延見込みです。
他タスクへの依存はありません。

## 対応案

A. リトライ間隔を調整して現行方式で対応（+1日）
B. Stripe の Bulk API に切り替え（+0.5日、ただし追加調査が必要）

## 相談事項

B案の方が早そうですが、Bulk API は本番未検証です。どちらで進めますか？
```

```
「この報告が優れている点は何ですか？」
```

受講者の回答を待ってからまとめ:

- **状況（What）**: 何が起きているかが具体的
- **影響（Impact）**: どのくらい影響があるかが定量的
- **対応案（Options）**: 選択肢が複数あり、それぞれのトレードオフが明記
- **相談事項（Ask）**: 相手に何を判断してほしいかが明確

---

## Step 3: 演習

```
「以下のシナリオから 1 つ選び、「良い報告」を書いてみてください:

A. 担当タスクが遅延しそうなとき
B. 技術選定で迷っているとき
C. バグを発見したとき

シナリオの詳細は自分で想定して構いません。
（実際の業務経験があれば、それを題材にしてください）」
```

受講者が書いた報告を以下の観点で評価:

| 観点              | チェック内容                                       | 評価  |
| ----------------- | -------------------------------------------------- | ----- |
| 状況（What）      | 何が起きているかが具体的か                         | ○/△/× |
| 影響（Impact）    | 影響範囲・程度が定量的か                           | ○/△/× |
| 対応案（Options） | 選択肢が複数あるか、トレードオフが明記されているか | ○/△/× |
| 相談事項（Ask）   | 相手に何を判断してほしいかが明確か                 | ○/△/× |

フィードバック後、改善版を一緒に作成する。

---

## Step 4: ProjSight との対応

```
「気づきましたか？
良い報告の構造は、ProjSight のエンティティと対応しています:

| 報告の要素 | ProjSight エンティティ |
|-----------|----------------------|
| 状況（What） | タスク description / Issue description |
| 影響（Impact） | Risk の impact / Issue の severity |
| 対応案（Options） | DR の alternatives |
| 相談事項（Ask） | Question |

AI への指示を構造化する力は、
そのまま人間へのコミュニケーション力です。

逆に言えば、ProjSight を日常的に使うことで、
報告・連絡・相談の構造化スキルも自然に磨かれます。」
```

## 注意事項

- 受講者の業務経験に応じてシナリオの難易度を調整する
- 「報告」だけでなく「連絡」「相談」にも構造化が有効であることを伝える
- ProjSight との対応関係は暗記ではなく「気づき」として提示する
- Wave 2（Context Engineering）への期待を高めて終わる
